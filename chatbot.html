Food Ordering Chatbot

Conversational assistant using NLP + FastAPI + MySQL .

Try Live Demo

Project Overview

This food-ordering chatbot transforms static hotel websites into a smart conversational interface. It uses Dialogflow (NLP), FastAPI (backend), and MySQL (database) to let customers place, track, and cancel orders in plain English—no dropdowns or forms.

Problem Statement

  • Static websites require users to manually browse menus.
  • Menu changes need developer effort and redeployment.
  • Support staff are often needed for tracking/canceling orders.

This leads to user friction and wasted business hours. We needed automation that feels human and scalable.

Solution Approach

Order Placement

Natural language interface to view menu, add/remove items, confirm orders.

Order Tracking

Real-time order status lookup via unique order ID from chat.

Architecture & Data Flow

Dialogflow handles intent detection & session context.
FastAPI acts as a middleware routing logic and commands.
MySQL persists order data with stored procedures for atomic transactions.

Key Features

Conversational UX

Graceful fallbacks and error handling improve user experience.

Dynamic Menu

Menu items are read directly from DB—no redeployment needed for updates.

Atomic Operations

Stored procedures ensure reliable add/remove with rollback on failure.

In-Memory Cart

Session-based cart storage lets users build orders before final confirmation.

Tech Stack
